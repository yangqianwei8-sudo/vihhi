# 🏗️ 项目架构说明

## 📋 项目架构类型

这个项目**主要使用Django后端渲染**，而不是完全的前后端分离架构。

### 实际架构

1. **Django后端（主要界面）**
   - 使用Django模板引擎渲染HTML页面
   - 提供完整的系统界面（工作台、项目中心、交付管理等）
   - 运行在：**8001端口**
   - **这是您应该使用的系统界面**

2. **Vue前端（占位/未完成）**
   - 目前只是一个简单的占位应用
   - Home.vue只显示欢迎文字，不是实际系统
   - 运行在：8080端口
   - **这不是实际的系统界面**

## ✅ 正确的访问方式

### 使用Django后端界面（推荐）

**访问地址**：
- **首页/工作台**: http://localhost:8001/
- **登录页**: http://localhost:8001/login/
- **交付管理**: http://localhost:8001/delivery/
- **创建交付单**: http://localhost:8001/delivery/create/

**特点**：
- ✅ 完整的系统功能
- ✅ 所有模块都已实现
- ✅ 包含交付管理、项目中心等所有功能
- ✅ 使用Django模板渲染，功能完整

### Vue前端（不推荐使用）

**访问地址**：
- http://localhost:8080/

**特点**：
- ❌ 只是占位页面
- ❌ 没有实际功能
- ❌ 只显示欢迎文字

## 🔍 为什么不同？

### Django后端界面（8001端口）
- 使用 `backend/templates/home.html` 模板
- 包含完整的系统功能：
  - 工作台仪表盘
  - 项目中心
  - 交付管理
  - 所有业务模块
- 这是**实际的生产系统**

### Vue前端界面（8080端口）
- 使用 `frontend/src/views/Home.vue` 组件
- 只是一个简单的欢迎页面：
  ```vue
  <template>
    <div class="home">
      <h1>维海科技信息化管理平台</h1>
      <p>欢迎使用维海科技信息化管理平台</p>
    </div>
  </template>
  ```
- **这不是实际的系统界面**

## 🎯 解决方案

### 方案1：使用Django后端（推荐）

**直接访问Django后端**：
```
http://localhost:8001/
```

这是完整的系统，包含所有功能。

### 方案2：完善Vue前端（如果需要）

如果需要使用Vue前端，需要：
1. 创建完整的Vue组件
2. 实现所有业务功能
3. 连接后端API
4. 这需要大量开发工作
